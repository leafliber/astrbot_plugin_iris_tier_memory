- 第一层：普通消息上下文缓冲（按群聊）
    - 简单少量的上下文消息，采用队列存储，固定长度
    - 在处理前全量加入上下文：统一标准消息格式
    - 最新的固定条消息会加入上下文
    - 超过固定的token，且当前队列消息数大于注入上下文所需的最小消息条数时，触发一次自动总结（提供开关），将消息量压缩到固定条上下文的长度；以"大于最小条数"为前提，避免单条消息本身token极大时陷入反复总结的死循环
    - 直接舍弃超过设定的token上限的单条消息，不让其入队
- 第一层补充层：LLM消息上下文（astrbot自带，但与记忆层少部分重复，且影响时间感知）
    - 作为即时短期记忆
    - 由astrbot自动管理
    - 提供选择，接入本插件自动控制LLM消息长度 summary，减少冗余和消耗
- 第二层：记忆库（ChromaDB）
    - 支持群聊ID隔离，设计特性：默认关闭隔离（全局共享），可按需开启；隔离与否均为合理使用场景，由用户根据部署需求自行选择
    - 带有去重逻辑过滤
    - 独立降级：ChromaDB不可用时，仅L2记忆检索降级失效，L1和L3不受影响，主流程继续运行
    - 冷启动兜底：新用户/新群聊首次触发检索时，L2无记忆可返回，直接以空记忆集继续处理，不报错；后续随消息总结任务逐步写入记忆
    - 响应延迟保护：L2检索设置超时上限（可配置，默认2000ms）；超时后跳过本层结果，直接进入L3或以空集合并，不阻塞主流程
    - 容量上限与淘汰策略
        - 每个 collection（人格/默认）设置最大条目数上限（可配置，默认10000条）
        - 超限时触发淘汰：按遗忘权重综合评分排序，优先淘汰评分最低的条目
        - 淘汰操作由定时任务统一执行，不在写入路径上触发，避免影响响应延迟
        - 淘汰前先执行一次合并，尝试将碎片记忆合并后再评估是否超限
- 第三层：知识图谱（KuzuDB）
    - 作为半永久记忆，只能由LLM从上下文和记忆中总结添加
    - 使用嵌入式 KuzuDB（无需独立服务进程，数据存放在 astrbot 插件数据目录的图谱子目录下）
    - 支持人格隔离，不开启时使用 default 图
    - 独立降级：KuzuDB不可用时，仅L3图谱检索降级失效，L1和L2不受影响，记忆增强模块自动跳过图增强步骤
    - 冷启动兜底：新用户/新群聊图谱为空时，直接返回空结果，不报错；随LLM总结任务逐步建立图结构
    - 响应延迟保护：L3检索设置超时上限（可配置，默认1500ms）；超时后跳过图增强结果，仅使用L2向量结果
    - 容量上限与淘汰策略
        - 设置节点数和边数上限（可配置）
        - 超限时按遗忘权重综合评分淘汰评分最低的节点及其关联边
        - 淘汰由定时任务执行，记录淘汰日志供调试
    - 遗忘权重算法（记忆重要性综合评分，同时适用于 L2 ChromaDB 条目和 L3 KuzuDB 节点）
        - 综合得分 S = w1·R + w2·F + w3·C + w4·(1 - D)，默认权重 w1=0.3, w2=0.3, w3=0.2, w4=0.2
        - R（近因性）：最近一次被访问/引用距今的时间衰减，衰减函数 R = exp(-λ·Δt)，λ 默认0.01/小时，即约3天后衰减至约0.5
        - F（频率性）：历史被访问/引用次数，归一化到 [0,1]，上限可配置（默认100次为满分）
        - C（置信度）：LLM写入时附带的置信评级（高/中/低对应1.0/0.6/0.3），可被用户修正操作更新
        - D（孤立度）：仅用于 L3 图谱节点，= 孤立边数/总边数；边越少孤立度越高，得分越低；L2条目此项固定为0
        - 得分低于阈值（可配置，默认0.2）且距上次访问超过保留期（可配置，默认30天）时标记为待淘汰，由定时任务批量删除
        - 权重、衰减率、阈值、保留期参数均通过 config 暴露为隐藏配置
        - 在最终遗忘时使用LLM最后兜底确认（可选）
- 图片解析（附加功能）
    - 解析模式
        - 全：所有图片
        - 相关：触发LLM时在上下文缓冲的、触发LLM的、被引用回复的、OCR命中关键词的
        - 关闭：默认模式
    - 每日解析限额
        - 限额粒度为全局（跨所有群聊和用户共享同一计数器）
        - 超限后静默跳过图片解析，降级为无图片上下文继续处理
        - 每日UTC 00:00自动重置计数器
    - 有图片去重（pHash）与无效图过滤（纯色/过小）机制
- 定时任务（辅助模块）
    - 清理与整理专项任务
        - 遗忘机制：定期计算记忆权重评分，批量清理评分低于阈值且超过保留期的条目
        - 合并机制：将碎片化相似记忆合并为长文本
        - 权重更新：基于互动频率批量更新知识图谱边权重和访问频率字段
        - 数据清洗：自动检测并标记低置信度数据
        - 写竞争保护：上述四项任务通过任务队列串行调度，同一时刻只有一项任务持有存储写锁；遗忘/合并操作对ChromaDB和KuzuDB分别加写锁，权重更新和数据清洗各自独立加锁，互不嵌套，防止死锁
    - 普通消息总结（按群分析）
        - 异步事件驱动：消息流入后异步触发，不阻塞主流程
        - 写入延迟保护：异步总结任务写入记忆前，先检查是否有正在进行的LLM检索；若有，则等待该检索完成后再写入，防止脏读窗口期
        - 总结内容：
            - 群聊画像更新（含长期标签）
            - 用户画像更新
            - 写入有价值记忆
- 画像（附加功能模块）
    - 群聊画像
        - 群聊ID
        - 群聊名称
        - 当前所聊话题（短期，随对话更新）
        - 群聊兴趣点（中期，跨多次对话沉淀）
        - 长期标签（核心特征，不易随单次对话改变，如"技术交流群""游戏群""摸鱼为主"）
        - 不可聊的点（黑名单话题，LLM应回避）
        - 活跃用户列表（近期发言较多的用户ID）
        - 群聊活跃时段（高频发言的时间段）
        - 群聊氛围标签（如"严肃""轻松""毒舌""温和"）
        - 常用语/群内梗（群内高频出现的特定表达或内部梗）
        - 其他（支持自定义字段）
    - 用户画像（支持群聊隔离检索；关闭隔离时，通过 default 访问/更新全局画像）
        - 用户ID（平台原始ID）
        - 历史曾用ID（平台改名记录，设计特性：用于跨改名识别同一用户）
        - 用户昵称/备注
        - 当前情感状态（设计特性：记录用户最近对话中流露的情绪倾向，如"愉快""焦虑""平静"，供LLM调整回应语气）
        - 性格标签（如"外向""理性""幽默""敏感"，由LLM从对话中归纳）
        - 兴趣爱好（用户提及或表现出兴趣的领域）
        - 职业/身份信息（用户主动透露的背景信息）
        - 常用语言风格（如"简洁""喜欢用表情""喜欢长篇大论"）
        - 对bot的称呼/关系设定（用户对bot的个性化称呼或角色定义）
        - 重要纪念日/提醒事项（用户提及的生日、重要日期等）
        - 禁忌话题（用户明确表示不想聊的内容）
        - 历史重要事件摘要（对用户影响较大的对话事件简述）
        - 最近一次互动时间
        - 其他（支持自定义字段）
    - 分析模式：全部消息、仅相关消息
    - 用户画像群聊隔离（设计特性：同一用户在不同群聊中可有不同画像，隔离关闭时合并为全局画像）
    - 增加版本控制：画像数据带版本号，避免读写竞争
- TOOL钩子（辅助模块）
    - 保存记忆
    - 读取记忆
    - 获取群聊画像
    - 获取用户画像
    - 修正：用户主动纠正幻觉（修正错误记忆/画像）
        - 修正操作同时更新ChromaDB向量条目和KuzuDB关联节点/边，保证两层存储一致
        - 若修正内容与知识图谱中现有节点冲突，标记该节点为"待复核"状态，由下次LLM总结决定是保留、覆盖还是删除
- 人格隔离（注意点说明，提供开关）：记忆、画像和人格绑定（可配置关闭）
    - 切换原子性：人格切换时同步切换ChromaDB collection命名空间和KuzuDB图命名空间，刷新上层缓存；未启用人格切换时，ChromaDB和KuzuDB均使用default命名空间
    - 切换操作加互斥锁，切换完成前阻塞检索和写入请求，防止切换过程中读到混合命名空间数据
- 记忆增强（注意点说明，提供开关）

    - 重排序增强：向量+图谱结果合并后，经轻量模型重排序，仅取 Top-K 注入上下文
    - 上下文长度保护：根据设置的Token预算，计算并控制注入记忆的数量，防止溢出

- 组件初始化（特殊点说明）
    - 插件首次加载时，按顺序检查并初始化所有组件
        - ChromaDB：检查collection是否存在，不存在则创建default collection及各人格collection框架
        - KuzuDB：检查数据库文件是否存在，创建default图及必要的节点/边schema（节点ID、权重、置信度、访问时间等字段），数据存放在astrbot插件数据目录的图谱子目录下
        - 画像存储：检查群聊画像和用户画像表/文件是否存在，不存在则以空结构初始化
        - 定时任务调度器：注册所有定时任务，首次注册时检查是否存在残留锁文件并清理
        - 图片解析限额计数器：读取持久化计数文件，若不存在则初始化为0
    - 各组件独立初始化，互不依赖：某一组件初始化失败只影响对应层的功能，其余组件继续正常启动并记录失败日志；全部组件均失败时，以仅L1模式运行，不影响astrbot主进程
